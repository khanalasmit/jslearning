About the Architecture

An overview of the design patterns, CRUD lifecycle, and Django concepts implemented.

CRUD Operations

How data flows through the application

  • Create: Store creation is handled by the chai_store view, which processes POST payloads submitted from the store addition form at /chai_stores/.
  • Read: Database records are queried and passed to templates via views like all_chai (listing chais), definite_chai (chai detail), and chai_store (which uses prefetch_related to list stores and their nested chai varieties efficiently).
  • Update: In-place updates are made via the asynchronous updating_chaistore view, receiving PATCH JSON requests from JavaScript to dynamically rename store entries on blur/enter.
  • Delete: Asynchronous record destruction is handled by the deleting_chaistore view, which executes database .delete() operations via DELETE requests when a store name is left blank.

Django Templating

Key template tags used to build the user interface

  • {% verbatim %}{% extends %}{% endverbatim %} & {% verbatim %}{% block %}{% endverbatim %}: Enforces template inheritance and the DRY (Don't Repeat Yourself) principle, utilizing a common layout from base.html.
  • {% verbatim %}{% for store in stores %}{% endverbatim %}: Iterates through Model QuerySets, powering our nested loops and table lists dynamically.
  • {% verbatim %}{{ variable }}{% endverbatim %}: Injects Python variable data directly into the DOM context (e.g., embedding the store.id into HTML data-attributes).
  • {% verbatim %}{% csrf_token %}{% endverbatim %}: A vital security tag injected into HTML forms to generate a hidden token protecting against forgery attacks.

Middleware Layers

The fundamental request/response processing hooks

  • CsrfViewMiddleware: Acts as a shield against Cross-Site Request Forgery. It enforces security by validating either the hidden form token or the X-CSRFToken HTTP header we appended to our JavaScript fetch requests.
  • SessionMiddleware & AuthenticationMiddleware: Standard Django stack middlewares that attach session attributes and the authenticated request.user to each incoming HTTP request payload.
  • CommonMiddleware: Automatically provides conveniences like resolving missing trailing slashes (which prevents URL mismatch bugs with our Javascript API endpoints).
  • BuiltMiddlewareCalculates the time required for laoding the page which is included in the middleware.py of the main project folder.
